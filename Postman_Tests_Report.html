المرحلة 2

تسجيل مستخدم جديد - POST /auth/register

POST /api/auth/register
❌ قبل
Body (raw JSON): { "name": "اسم المستخدم", "email": "user@example.com", // إجباري "password": "password123", "password_confirmation": "password123", "phone": "09XXXXXXXX", // سوري فقط "role": "user" // إجباري } Response: "message": "تم التسجيل بنجاح. لم يتم توثيق حسابك بعد..."
✅ بعد
Body (raw JSON): { "name": "اسم المستخدم", "email": "user@example.com", // اختياري "password": "password123", "password_confirmation": "password123", "phone": "+963XXXXXXXX" // أي تنسيق } Response: "message": "تم التسجيل بنجاح"
🔄 التغييرات:
  • email: من إجباري ← اختياري
  • phone: من سوري فقط ← أي تنسيق (min:8, max:20)
  • role: محذوف (الجميع مستخدم عادي)
  • رسالة النجاح: بدون ذكر التوثيق
POST /api/auth/login
❌ قبل
Body (raw JSON): { "email": "user@example.com", // email فقط "password": "password123" } Response (فشل): if (email_verified_at == null) { "message": "لم يتم توثيق حسابك بعد..." }
✅ بعد
Body (raw JSON) - الخيار 1: { "email": "user@example.com", // بريد "password": "password123" } Body (raw JSON) - الخيار 2: { "email": "0912345678", // أو هاتف "password": "password123" } Body (raw JSON) - الخيار 3: { "email": "+963912345678", // هاتف مع رمز الدولة "password": "password123" } Response (فشل): // لا يوجد تحقق من التوثيق "message": "بيانات الاعتماد غير صحيحة"
🔄 التغييرات:
  • email يقبل: بريد إلكتروني أو رقم هاتف
  • إزالة التحقق من email_verified_at
  • إزالة رسالة "لم يتم توثيق حسابك"
المرحلة 4

تغيير البريد الإلكتروني - PUT /auth/email

PUT /api/auth/email
❌ قبل - غير موجود

هذا الـ API غير موجود في النظام القديم

✅ بعد - جديد
Headers: { "Authorization": "Bearer {token}" } Body (raw JSON): { "email": "new@example.com", "password": "current_password" } // نجاح { "success": true, "message": "تم تغيير البريد الإلكتروني بنجاح" } // فشل { "success": false, "message": "كلمة المرور غير صحيحة أو البريد مستخدم مسبقاً" }
POST /api/auth/change-password
❌ قبل
// لا تغيير Headers: { "Authorization": "Bearer {token}" } Body (raw JSON): { "current_password": "old_password", "password": "new_password", "password_confirmation": "new_password" }
✅ بعد
// لا تغيير - كما هو Headers: { "Authorization": "Bearer {token}" } Body (raw JSON): { "current_password": "old_password", "password": "new_password", "password_confirmation": "new_password" }
📝 ملاحظة: هذا الـ API remained unchanged as per request.
PUT /api/auth/profile
❌ قبل
Body (raw JSON): { "name": "اسم جديد", "phone": "09XXXXXXXX" // سوري فقط }
✅ بعد
Body (raw JSON): { "name": "اسم جديد", "phone": "+963XXXXXXXX", // أي تنسيق "email": "new@example.com" // جديد }
🔄 التغييرات:
  • phone: أي تنسيق مقبول
  • email: يمكن تغييره
المرحلة 5

إضافة حي جديد - POST /regions

POST /api/regions
❌ قبل - Admin فقط
// URL: /api/admin/regions Headers: { "Authorization": "Bearer {admin_token}" // أدمن فقط } Body (raw JSON): { "name": "حي الزهراء", "type": "neighborhood", "parent_id": 123 }
✅ بعد - مستخدم عادي
// URL: /api/regions Headers: { "Authorization": "Bearer {user_token}" // أي مستخدم } Body (raw JSON): { "name": "حي الزهراء", "type": "neighborhood", "parent_id": 123 }
🔄 التغييرات:
  • URL تغير: /api/admin/regions → /api/regions
  • الصلاحية: Admin فقط → أي مستخدم مسجل
مراحل متعددة

❌ APIs محذوفة

Method Endpoint السبب
POST /api/auth/promote-to-admin إلغاء دور مالك العقار (owner)
POST /api/auth/verify-email إلغاء توثيق البريد
POST /api/auth/send-verification-email إلغاء توثيق البريد
GET /api/admin/users/unverified إلغاء توثيق البريد
POST /api/admin/users/{id}/verify إلغاء توثيق البريد

📊 ملخص المقارنة

API التغيير
POST /auth/register email اختياري، phone أي تنسيق، role محذوف
POST /auth/login يقبل email أو phone، إزالة التحقق من التوثيق
PUT /auth/profile إضافة email، phone أي تنسيق
PUT /auth/email جديد - تغيير البريد
POST /auth/change-password لا تغيير
DELETE /auth/account جديد - حذف الحساب
POST /regions متاح للمستخدم العادي (ليس أدمن فقط)
