📖 نظرة عامة

💡 ما هي Dashboard APIs؟

واجهات برمجة تطبيقات لوحة التحكم توفر للأدمن رؤية شاملة ومفصلة لجميع البيانات في التطبيق. تسمح هذه الواجهات بالحصول على إحصائيات وتقارير وبيانات تحليلية مهمة.

🎯 الهدف من لوحة التحكم

  • مراقبة البيانات: تتبع المستخدمين والعقارات وطلبات التواصل
  • تقارير إحصائية: الحصول على أرقام وإحصائيات دقيقة
  • تحليل الاتجاهات: فهم أنماط النمو والتوزيع
  • اتخاذ القرارات: دعم القرارات بناءً على البيانات

🏗️ البنية التقنية

Request → DashboardRequest → AdminController → AdminService → Resource → ApiResponse

تتبع الواجهات البنية التقنية التالية:

  • Request: التحقق من صحة الطلب ومدخلات المستخدم
  • Controller: استقبال الطلب وتنسيق الاستجابة
  • Service: المنطق التجاري ومعالجة البيانات
  • Resource: تنسيق وتحويل البيانات للإرجاع
  • ApiResponse: صيغة الاستجابة الموحدة

🔐 المصادقة والترخيص

⚠️ متطلب أساسي

جميع واجهات Dashboard APIs تتطلب صلاحيات الأدمن. يجب إرسال Token المصادقة مع كل طلب.

Header الطلب

HTTP Headers مطلوب 
Authorization: Bearer YOUR_ADMIN_TOKEN
Accept: application/json
Content-Type: application/json

كيفية الحصول على Token

  1. سجل الدخول كأدمن عبر POST /api/auth/login
  2. سيتم إرجاع Token في حقل token
  3. استخدم هذا Token في Header Authorization
مثال على تسجيل الدخول
POST /api/auth/login

{
    "email": "admin@example.com",
    "password": "your-password"
}

📦 صيغة الاستجابة الموحدة

جميع واجهات API ترجع استجابة موحدة لضمان التناسق وتسهيل المعالجة من الجهة المستهلكة.

✅ الاستجابة الناجحة

HTTP 200 OK
{
    "success": true,
    "message": "تمت العملية بنجاح",
    "data": {
        // البيانات المطلوبة حسب الواجهة
    }
}

❌ الاستجابة الخاطئة

HTTP 4xx/5xx
{
    "success": false,
    "message": "رسالة الخطأ",
    "errors": null
}

شرح الحقول

الحقل النوع الوصف
success Boolean يحدد نجاح أو فشل العملية
message String رسالة توضيحية عن نتيجة العملية
data Object/Array البيانات المطلوبة (فارغة في حالة الخطأ)
errors Object تفاصيل الأخطاء (في حالة وجودها)

📊 1. الإحصائيات الشاملة

GET /admin/statistics

ترجع هذه الواجهة إحصائيات شاملة ومجمعة عن المستخدمين والعقارات وطلبات التواصل.

رأس الطلب (Headers)

Accept: application/json

جسم الطلب (Query Parameters)

هذه الواجهة لا تتطلب أي معلمات - طلب GET بسيط

HTTP 200 OK
{

    "message": "تمت العملية بنجاح",
    "data": {
        "users": {
            "total": 150,
            "owners": 45,
            "unverified": 12,
            "banned": 3
        },
        "properties": {
            "total": 320,
            "today": 5,
            "active": 250,
            "rented": 30,
            "sold": 20,
            "pending": 15,
            "rejected": 5
        },
        "contact_requests": {
            "total": 85,
            "pending": 20
        }
    }
}

📋 5. ملخص العقارات

GET /admin/dashboard/properties-summary

ترجع ملخصاً إحصائياً للعقارات مع إمكانية الفلترة حسب النوع (الكل، المباعة، المؤجرة).

رأس الطلب (Headers)

Required Headers
Authorization: Bearer {admin_token}
Accept: application/json

معاملات الاستعلام (Query Parameters)

المعامل النوع القيمة الافتراضية الوصف
type String all نوع الفلترة: all, sold, rented
from_date String (Date) null تاريخ البداية للفلترة (اختياري)
to_date String (Date) null تاريخ النهاية للفلترة (اختياري)

أمثلة على الاستخدام

جميع العقارات
GET /api/admin/dashboard/properties-summary
العقارات المباعة فقط
GET /api/admin/dashboard/properties-summary?type=sold
العقارات المؤجرة مع فلترة التاريخ
GET /api/admin/dashboard/properties-summary?type=rented&from_date=2024-01-01&to_date=2024-12-31

✅ مثال على استجابة صحيحة

HTTP 200 OK
{
    "success": true,
    "message": "تمت العملية بنجاح",
    "data": {
        "total_count": 320,
        "total_price": 156500000,
        "avg_price": 489063,
        "filters": {
            "from_date": null,
            "to_date": null,
            "type": "all"
        }
    }
}

⚠️ أكواد الأخطاء الشائعة

كود HTTP الرسالة السبب المحتمل
401 غير مصرح Token غير صالح أو منتهي الصلاحية
403 غير مصرح لك بهذه العملية المستخدم ليس أدمن أو ليس لديه صلاحيات
404 غير موجود المورد المطلوب غير موجود
422 بيانات غير صالحة فشل في التحقق من صحة المدخلات
500 خطأ في الخادم مشكلة في الخادم، راجع console

❌ مثال: Unauthorized

HTTP 401 Unauthorized
{
    "success": false,
    "message": "غير مصرح",
    "errors": null
}

❌ مثال: Forbidden

HTTP 403 Forbidden
{
    "success": false,
    "message": "غير مصرح لك بهذه العملية",
    "errors": null
}

❌ مثال: Validation Error

HTTP 422 Unprocessable Entity
{
    "message": "The given data was invalid.",
    "errors": {
        "field_name": [
            "حقل الاسم مطلوب"
        ]
    }
}
