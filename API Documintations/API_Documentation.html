📝 آخر التحديثات

Updates

📋 التغييرات في النسخة 2.0.0:

  • إزالة دور "مالك العقار" - أي مستخدم مسجل يمكنه إضافة عقارات
  • إضافة إمكانية تغيير البريد الإلكتروني (PUT /auth/email)
  • إضافة إمكانية حذف الحساب (DELETE /auth/account)
  • تسجيل الدخول بالبريد الإلكتروني أو رقم الهاتف
  • إزالة التحقق من البريد الإلكتروني عند تسجيل الدخول
  • تحديث صلاحيات رؤية العقارات: الأدمن يرى الكل، المستخدم يرى نشطاته + نشط
  • تحديث طلبات التواصل: المالك يرى طلباته المقبولة فقط
  • إضافة villa كنوع عقار
  • إزالة route ترقية الأدمن

🔐 المصادقة

Auth
POST /auth/register عام
تسجيل مستخدم جديد

📝 ملاحظة

أي مستخدم جديد يمكنه إضافة عقارات. البريد الإلكتروني اختياري.

Body

المعاملالنوعالوصف
name *stringاسم المستخدم
emailstringالبريد الإلكتروني (اختياري)
password *stringكلمة المرور (min: 8)
password_confirmation *stringتأكيد كلمة المرور
phone *stringرقم الهاتف

مثال

{ "name": "أحمد محمد", "email": "ahmed@example.com", "password": "password123", "password_confirmation": "password123", "phone": "0912345678" }

استجابة (201)

{ "success": true, "message": "تم التسجيل بنجاح", "data": { "user": { "id": 1, "name": "أحمد محمد", "email": "ahmed@example.com", "phone": "0912345678", "is_admin": false }, "token": "1|abc123xyz..." } }
POST /auth/login عام
تسجيل الدخول (بالبريد أو الهاتف)

📝 ملاحظة

يمكن تسجيل الدخول بالبريد الإلكتروني أو رقم الهاتف.

Body

المعاملالنوعالوصف
email *stringالبريد الإلكتروني أو رقم الهاتف
password *stringكلمة المرور

مثال

{ "email": "0912345678", "password": "admin123" }

استجابة (200)

{ "success": true, "message": "تم تسجيل الدخول بنجاح", "data": { "user": { "id": 1, "name": "مدير النظام", "is_admin": true }, "token": "2|xyz789abc..." } }

خطأ - حساب محظور (403)

{ "success": false, "message": "الحساب محظور. يرجى التواصل مع الإدارة" }

Body

🏠 العقارات

Properties

📌 صلاحيات رؤية العقارات

الزائر: يرى العقارات النشطة فقط
المستخدم: يرى العقارات النشطة + عقاراته (كل الحالات)
الأدمن: يرى جميع العقارات بجميع الحالات

GET /properties عام
جلب قائمة العقارات النشطة

Query Parameters

المعاملالنوعالوصف
typestringsale أو rent
property_typestringapartment, house, villa, shop, land, farm, store
region_idintegerمعرف المنطقة
searchstringكلمة البحث
min_pricenumberالسعر الأدنى
max_pricenumberالسعر الأقصى
per_pageintegerعدد بالصفحة (افتراضي: 20)

استجابة (200)

{ "success": true, "data": { "data": [{ "id": 1, "title": "شقة فاخرة في المزة", "price": "85000000.00", "type": "sale", "images": [...], "owner": { "id": 1, "name": "مكتب المفتاح" } }], "meta": { "current_page": 1, "last_page": 5, "per_page": 20, "total": 100 } } }
GET /properties/types عام
جلب أنواع العقارات

استجابة (200)

{ "success": true, "data": [ {"value": "apartment", "label": "شقة"}, {"value": "house", "label": "منزل"}, {"value": "villa", "label": "فيلا"}, {"value": "shop", "label": "محل"}, {"value": "land", "label": "أرض"}, {"value": "farm", "label": "مزرعة"}, {"value": "store", "label": "متجر"} ] }
GET /properties/{id} عام
جلب تفاصيل عقار

📌 قواعد العرض

العقار نشط: الجميع
العقار غير نشط: المالك أو الأدمن فقط

POST /properties يتطلب توثيق
إضافة عقار جديد

📌 ملاحظة

أي مستخدم مسجل يمكنه إضافة عقارات. العقار يُنشأ بحالة pending حتى يوافق الأدمن.

Body

المعاملالنوعالوصف
title *stringعنوان العقار
description *stringوصف العقار
price *numericالسعر
type *stringsale أو rent
property_type *stringapartment, house, villa, shop, land, farm, store
area *numericالمساحة (م²)
region_id *integerمعرف الحي
location *stringالموقع النصي
latitudenumericخط العرض
longitudenumericخط الطول
images[]fileصور (max: 10)

استجابة (201)

{ "success": true, "message": "تم إنشاء العقار بنجاح", "data": { "id": 16, "title": "شقة جديدة", "status": "pending" } }
GET /properties/my-properties يتطلب توثيق
جلب عقارات المستخدم (جميع الحالات)

📌 ملاحظة

يُرجع جميع عقارات المستخدم بما فيها المعلقة والمرفوضة.

PUT /properties/{id} يتطلب توثيق
تعديل عقار (المالك أو الأدمن)

📌 ملاحظة

المستخدم يمكنه تعديل عقاراته فقط. الأدمن يمكنه تعديل أي عقار.

DELETE /properties/{id} يتطلب توثيق
حذف عقار (المالك أو الأدمن)
POST /properties/{id}/rented يتطلب توثيق
تعليم العقار كمؤجر
POST /properties/{id}/sold يتطلب توثيق
تعليم العقار كمباع

📞 التواصل

Contact

📱 كيف يعمل نظام التواصل

1. المستخدم يرسل طلب تواصل → يُرسل إشعار للأدمن
2. الأدمن يوافق/يرفض الطلب
3. إذا وافق الأدمن → المستخدم يرى رقم المالك

POST /contact يتطلب توثيق
إرسال طلب تواصل

Body

المعاملالنوعالوصف
property_id *integerمعرف العقار
name *stringالاسم
phone *stringرقم الهاتف
messagestringرسالة إضافية

استجابة (201)

{ "success": true, "message": "تم إرسال طلب التواصل بنجاح", "data": { "id": 1, "property_id": 1, "status": "pending" } }
GET /contact/my-requests يتطلب توثيق
طلباتي المرسلة
GET /contact/my-received يتطلب توثيق
الطلبات المستلمة على عقاراتي (المقبولة فقط)

📌 ملاحظة

يُرجع فقط الطلبات المقبولة من الأدمن. يمكن تمرير status للتصفية.

GET /contact/check/{propertyId} يتطلب توثيق
فحص حالة الطلب

استجابة - معتمد (200)

{ "success": true, "data": { "has_request": true, "status": "approved", "created_at": "2026-03-26T12:00:00Z", "owner": { "name": "مكتب المفتاح", "phone": "0912345678" } } }
