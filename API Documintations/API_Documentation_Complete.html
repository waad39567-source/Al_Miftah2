Overview

🎯 ما هو نظام المفتاح؟

نظام المفتاح هو منصة متكاملة لإدارة العقارات في سوريا. يتيح للمستخدمين البحث عن العقارات، والتواصل مع المالكين، وإدارة العقارات. النظام يدعم أنواع متعددة من المستخدمين بأدوار وصلاحيات مختلفة.

🏗️ هيكل النظام

يعتمد النظام على بنية REST API مع authentication عبر Laravel Sanctum. جميع البيانات تُرجع بصيغة JSON مع هيكل موحد للاستجابات.

📝 آخر التحديثات (مارس 2026)
  • إزالة دور مالك العقار - أي مستخدم مسجل يمكنه إضافة عقارات
  • إضافة إمكانية تغيير البريد الإلكتروني
  • إضافة إمكانية حذف الحساب
  • تحديث صلاحيات عرض العقارات: الأدمن يرى الكل، المستخدم يرى نشطاته + نشط
  • تحديث صلاحيات طلبات التواصل: أي مستخدم يمكنه رؤية طلباته المقبولة

📊 Base URL

http://127.0.0.1:8000/api

📦 هيكل الاستجابة الموحد

استجابة ناجحة

{ "success": true, "message": "رسالة النجاح", "data": { /* البيانات المطلوبة */ } }

استجابة خطأ

{ "success": false, "message": "رسالة الخطأ", "errors": { /* أخطاء التحقق إن وجدت */ } }

🔐 المصادقة وإدارة الحسابات

Authentication

🔑 نظام المصادقة

يعتمد النظام على Laravel Sanctum للمصادقة. عند تسجيل الدخول يتم إصدار Token يستخدم في جميع الطلبات اللاحقة. يجب إرسال Token في Header مع كل طلب محمي.

📌 طريقة إرسال Token
Authorization: Bearer {token}
POST /auth/register عام
تسجيل مستخدم جديد في النظام
💡 متى يُستخدم؟

عندما يريد شخص جديد إنشاء حساب في النظام. يمكنه التسجيل بدون بريد إلكتروني.

الـ Header

Content-Type: application/json

الـ Body

المعاملالنوعالوصف
name *stringاسم المستخدم
email اختياريstringالبريد الإلكتروني (فريد إن تم إدخاله)
password *stringكلمة المرور (min: 8)
password_confirmation *stringتأكيد كلمة المرور
phone *stringرقم الهاتف

مثال الطلب

{ "name": "أحمد محمد", "email": "ahmed@example.com", "password": "password123", "password_confirmation": "password123", "phone": "0912345678" }
201 Created تم التسجيل بنجاح
{ "success": true, "message": "تم التسجيل بنجاح", "data": { "user": { "id": 1, "name": "أحمد محمد", "email": "ahmed@example.com", "phone": "0912345678", "is_admin": false }, "token": "1|abc123..." } }
POST /auth/login عام
تسجيل الدخول والحصول على Token
💡 متى يُستخدم؟

للتعرف على المستخدم وإصدار Token. يمكن تسجيل الدخول بالبريد الإلكتروني أو رقم الهاتف.

الـ Body

المعاملالنوعالوصف
email اختياريstringالبريد الإلكتروني أو رقم الهاتف
password *stringكلمة المرور

مثال الطلب

{ "email": "0912345678", "password": "admin123" }
200 OK تم تسجيل الدخول
{ "success": true, "message": "تم تسجيل الدخول بنجاح", "data": { "user": { "id": 1, "name": "مدير النظام", "is_admin": true }, "token": "2|xyz789..." } }
401 بيانات خاطئة
{ "success": false, "message": "بيانات الاعتماد غير صحيحة" }
403 حساب محظور أو غير نشط
{ "success": false, "message": "الحساب محظور. يرجى التواصل مع الإدارة" }

🏠 إدارة العقارات

Properties

🏠 نظام العقارات

العقار هو الوحدة الأساسية في النظام. أي مستخدم مسجل يمكنه إضافة عقارات. كل عقار له مالك ويحتوي على: عنوان، وصف، سعر، نوع، موقع، صور.

📌 صلاحيات رؤية العقارات

الزائر: يرى العقارات النشطة فقط
المستخدم: يرى العقارات النشطة + عقاراته (بجميع حالاتها)
الأدمن: يرى جميع العقارات بجميع الحالات

📞 نظام التواصل

Contact

📞 كيف يعمل التواصل؟

يتيح هذا النظام للمستخدمين التواصل مع مالكي العقارات بشكل آمن:
1. يرسل المستخدم طلب تواصل يتضمن اسمه ورقم هاتفه
2. يُرسل الإشعار للأدمن
3. يوافق الأدمن على الطلب
4. يرى المالك رقم هاتف المستثمر

📱 إشعارات Push (FCM)

FCM

📱 ما هو FCM؟

Firebase Cloud Messaging (FCM) يتيح إرسال إشعارات Push للأجهزة. يجب على المستخدم حفظ Token الخاص بجهازه ليستقبل الإشعارات.

POST /fcm/token يتطلب توثيق
حفظ FCM Token للجهاز

الـ Body

{ "token": "fcm_device_token_here", "device_type": "android" }
DELETE /fcm/token يتطلب توثيق
حذف FCM Token (عند تسجيل الخروج)
POST /admin/send-notification أدمن فقط
إرسال إشعار Push لمستخدم معين

الـ Body

{ "user_id": 1, "title": "عنوان الإشعار", "body": "محتوى الإشعار", "extra": { "type": "custom_type", "id": 123 } }
