POST /auth/register عام

📝 الوصف

تسجيل مستخدم جديد في النظام. يتم إنشاء حساب غير موثق في البداية ويتطلب توثيق البريد الإلكتروني.

📋 Headers

المفتاحالقيمة
Content-Typeapplication/json
Acceptapplication/json

📦 Request Body

الحقل النوع مطلوب الوصف القواعد مثال
name string نعم الاسم الكامل للمستخدم required, max:255 "أحمد محمد"
email string نعم البريد الإلكتروني (يجب أن يكون فريد) required, email, unique "ahmed@test.com"
password string نعم كلمة المرور required, min:8, confirmed "password123"
password_confirmation string نعم تأكيد كلمة المرور required, same:password "password123"
phone string نعم رقم الهاتف (يجب أن يكون فريد) required, unique, max:20 "966501234567"
role string لا دور المستخدم in:user,owner "user"

✅ استجابة ناجحة

201 Created
{ "success": true, "message": "تم التسجيل بنجاح. لم يتم توثيق حسابك بعد يرجى التحقق من بريدك الإلكتروني لتوثيق الحساب", "data": { "user": { "id": 1, "name": "أحمد محمد", "email": "ahmed@test.com", "phone": "966501234567", "role": "user", "is_active": true, "created_at": "2026-03-09T10:30:00" } } }

❌ استجابات خاطئة

422 Unprocessable

1. بريد إلكتروني مكرر:

{ "success": false, "message": "Validation error", "errors": { "email": ["البريد الإلكتروني مستخدم من قبل"] } }

2. رقم هاتف مكرر:

{ "success": false, "message": "Validation error", "errors": { "phone": ["رقم الهاتف مستخدم من قبل"] } }

3. كلمة مرور غير متطابقة:

{ "success": false, "message": "Validation error", "errors": { "password": ["تأكيد كلمة المرور غير متطابق"] } }
POST /auth/login عام

📝 الوصف

تسجيل الدخول للحصول على token المصادقة. يُرجع token يستخدم في جميع الطلبات المحمية.

📦 Request Body

الحقلالنوعمطلوبالقواعدمثال
emailstringنعمrequired, email"ahmed@test.com"
passwordstringنعمrequired"password123"

✅ 200 - نجاح

{ "success": true, "message": "تم تسجيل الدخول بنجاح", "data": { "user": { "id": 1, "name": "أحمد محمد", "email": "ahmed@test.com", "phone": "966501234567", "role": "user", "is_active": true, "created_at": "2026-03-09T10:30:00" }, "token": "1|abc123def456..." } }

❌ استجابات خاطئة

401 Unauthorized 403 Forbidden

1. بيانات غير صحيحة:

{"success": false, "message": "بيانات الاعتماد غير صحيحة"}

2. حساب غير موثق:

{"success": false, "message": "لم يتم توثيق حسابك بعد يرجى التحقق من بريدك الإلكتروني لتوثيق الحساب"}

3. حساب غير نشط:

{"success": false, "message": "الحساب غير نشط"}
POST /auth/send-verification-email عام
⚠️ ملاحظة مهمة: هذا الـ API يضيف البريد الإلكتروني إلى Queue (قائمة الانتظار). يجب تشغيل php artisan queue:work لإرسال البريد فعلياً. رابط التوثيق يُرسل للمستخدم عبر البريد الإلكتروني.

📦 Request Body

الحقلالنوعمطلوبمثال
emailstringنعم"ahmed@test.com"

✅ 200 - نجاح

{ "success": true, "message": "تم إضافة إرسال رابط توثيق البريد الإلكتروني إلى قائمة الانتظار", "data": null }

❌ 400 - البريد موثق مسبقاً

{"success": false, "message": "تم توثيق البريد الإلكتروني مسبقاً"}

❌ 422 - البريد غير موجود

{ "success": false, "message": "Validation error", "errors": {"email": ["The selected email is invalid."]} }
