📝 أولاً: API ستتغير (تعديل)

# Method Endpoint التغييرات
1 POST /auth/register تعديل
  • إزالة email الإجباري ← اختياري
  • إبقاء phone إجباري
  • إزالة role من المدخلات
2 POST /auth/login تعديل
  • يقبل phone أو email مع password
3 PUT /auth/profile تعديل
  • إضافة إمكانية تغيير email
  • إضافة password للتحقق
4 PUT /auth/password تعديل
  • يتغير من POST إلى PUT
  • اسم الدالة: changePassword → updatePassword

🗄️ خامساً: تغييرات قاعدة البيانات

جدول users

الحقل التغيير السبب
email nullable لم يعد إجباري
phone required + unique مطلوب للتسجيل والدخول
أي تنسيق مقبول (varchar: min:8, max:20)
email_verified_at nullable إلغاء توثيق البريد
phone_verified_at nullable إلغاء التحقق من الهاتف
provider nullable لحساب Google (google/null)
provider_id nullable Google User ID

Cascade Delete للعلاقات

// في Migration أو Model // User → Properties $table->foreignId('owner_id') ->nullable() ->constrained('users') ->onDelete('cascade'); // ← جديد // User → ContactRequests $table->foreignId('user_id') ->nullable() ->constrained('users') ->onDelete('cascade'); // ← جديد // User → PropertyFavorites $table->foreignId('user_id') ->nullable() ->constrained('users') ->onDelete('cascade'); // ← جديد

⚙️ سادساً: تغييرات الخدمات (Services)

AuthService

الدالة التغيير
register()
  • إزالة التحقق من role
  • جعل email اختياري
  • إزالة إرسال بريد التوثيق
login()
  • قبول phone أو email
  • إزالة التحقق من email_verified_at

  • إزالة التحقق من البريد
